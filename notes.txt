=========================== New machine setup ================================

IMPORTANT: Before the first run, apply various required system configuration changes. (Read the script for documentation about what changes are applied)
  sudo system-config/apply-tegra-system-config.sh

Don't forget to set up the submodules after cloning:
  git submodule update --init --recursive

Install prerequisite packages:
  sudo apt install cmake scons libhidapi-dev libhidapi-hidraw0 libboost-all-dev libudev-dev clang lldb ninja-build libusb-1.0-0-dev libepoxy-dev libeigen3-dev libsdl2-dev libsocketcan-dev libeigen3-dev libceres-dev

Install the Jetson Multimedia API and other required libraries (CUDA for Tegra):
  sudo apt install nvidia-l4t-jetson-multimedia-api nvidia-cuda-dev

Build and install OpenCV 4.10 with some extra opencv_contrib modules. (TODO: more detailed directions)
  # Prereqs
  sudo apt install cmake ninja-build libpng-dev libtiff-dev libfreetype6-dev libavcodec-dev libavformat-dev libavutil-dev libswresample-dev libswscale-dev libtbb-dev
  # Git repositories
  git clone https://github.com/opencv/opencv.git -b 4.10.0
  git clone https://github.com/opencv/opencv_contrib -b 4.10.0
  # Make a build directory and run Cmake to configure
  mkdir opencv-build; cd opencv-build
  cmake -G Ninja -DOPENCV_EXTRA_MODULES_PATH=/path/to/opencv_contrib/modules -DWITH_CUDA=ON -DBUILD_LIST=aruco,calib3d,core,cudafilters,cudaimgproc,cudastereo,cudawarping,cudev,imgproc,imgcodecs,objdetect,stereo -DCUDA_ARCH_BIN="8.7" /path/to/opencv
  # Build and install
  ninja && sudo ninja install

Build the embedded Monado library required for distortion processing
  ./build-monado.sh

Optional: If you want to build the DepthAI backend, run the included script to build required libraries first:
  ./build-depthai-core.sh

Finally, build the project:
  scons



==============================================================================


If using the DepthAI backend, you might need to increase the usbfs buffer memory limit. The default of 16MB can be exhausted when using multiple devices.
  - Either set it via kernel boot parameter (in /boot/extlinux/extlinux.conf): usbcore.usbfs_memory_mb=128
  - Or add a startup task to run: sudo sh -c 'echo 128 > /sys/module/usbcore/parameters/usbfs_memory_mb'


Camera ISP settings are stored at /var/nvidia/nvcam/settings/camera_overrides.isp
The vendor ISP settings may be adjusted to improve flicker correction. These settings have been found to work well for capturing at 90fps and correcting for 60hz flicker:
flicker.ConfidenceThreshold = 32;
flicker.SuccessFrameCount = 8;
flicker.FailureFrameCount = 3;
flicker.CorrectionFreqListEntries = 4;
flicker.CorrectionFreqList = {60, 90, 100, 120};

The default Sharpening filter should be disabled to improve accuracy of Charuco target detection:
sharpness.v5.enable = FALSE;



====== Nvidia Visual Profiler (nvvp) (on desktop) =====
CUDA 11.1 + Ubuntu 20.04 nvvp will fail to launch out-of-the-box with a Java error.
solution: nvvp needs to run under the java 8 runtime (and as root, typically)

  sudo apt install openjdk8-jre
  sudo update-alternatives --config java
  select OpenJDK 8

nvvp can then be run with sudo /usr/local/cuda/bin/nvvp

Sample invocation to generate an nvprof tracefile:

(as root, on device)
  cd /home/hmdcam/hmdcam
  nvprof --system-profiling on --timeout 15 --cpu-profiling on --export-profile /tmp/dgpu-worker.nvvp build/bin/dgpu-worker

Sample invocation for profiling hmdcam + dgpu-worker together. Requires CWD to be correct so hmdcam can load shaders.
(as root, on device)
  cd /home/hmdcam/hmdcam
  nvprof --profile-child-processes --system-profiling on --timeout 15 --cpu-profiling on --export-profile /tmp/hmdcam-%p.nvvp build/bin/hmdcam


Copy /tmp/dgpu-worker.nvvp back to the host and load it into nvvp.
